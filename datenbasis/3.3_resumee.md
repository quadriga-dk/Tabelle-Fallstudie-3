# Resümee

```{admonition} Key points des Kapitels
:class: keypoint

**Datenbasis:** Rohdaten liegen selten in einer Form vor, die sofort analysiert werden kann. Die kritische Sichtung der Datenstruktur ist die Voraussetzung für jede weitere Verarbeitung.

**Datenaufbereitung:** Durch das Filtern, Bereinigen und Zusammenführen verschiedener Datensätze werden isolierte Informationen in einen auswertbaren, räumlichen Kontext gesetzt.

```

In diesem Kapitel haben Sie erfolgreich den Grundstein für das Gieß-den-Kiez-Dashboard gelegt. Dieser Schritt ist bei allen Projekten, in denen mit Daten gearbeitet wird, essentiell, da Sie sich nur mit einem stabilen Fundament auch wirklich sicher sein können, dass Sie mit Ihren Operationen auf den Daten auch das Ergebnis bekommen, dessen Berechnung Sie sich im Vorhinein als Ziel gesetzt hatten.

**Was haben Sie erreicht?**
* Sie haben die rohen Bewässerungsdaten (`giessdenkiez_bewässerungsdaten.csv`) sowie die Geodaten der Stadt Berlin (`bezirksgrenzen.geojson`) erfolgreich geladen.
* Sie haben gelernt, wie man diese Datensätze bereinigt, strukturiert und miteinander verknüpft, um sie für eine visuelle Analyse nutzbar zu machen.

**Wie geht es weiter?**
Mit Ihrem aufbereiteten Datensatz im Gepäck wechseln Sie nun von der semantischen zur syntaktischen Vorbereitung. Im nächsten Kapitel lernen Sie mehr über Dashboards im Allgemeinen, welche Werkzeuge zur Visualierung existieren und worin ihre Unterschiede liegen. Abschließend werden Sie das Gerüst Ihres Dashboards aufbauen.
